# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** `tests/Schema/ServerSettingsSchemaTest.php` hard-locks the exact settings key set: it has BOTH a `propertyProvider()` list AND `assertCount(N)`. Adding/removing a key in `schemas/server-settings.schema.json` fails the suite until you update the provider row AND bump the count (e.g. 18→19). The failure shows as a diff of expected-vs-actual key arrays at the `assertCount` line.
- **[gotcha]** `tests/Schema/SchemaPathsTest.php` is the lock-in test for the bundled `schemas/*.json` contract — when you add a new schema file or new schema fields/enum values, add the matching `SchemaPaths::xxx()` resolver + assertions there (it asserts filenames, the `type` enum, and required property keys).
- **[gotcha]** PHPStan L9 in this repo (config forbids `@var`, `@phpstan-ignore`, `assert()`, and casts to silence errors). Chained array access on a `json_decode(..., true)` result trips `Cannot access offset 'X' on mixed`. Fix by narrowing one level at a time: assign each nested value to a local var and call `$this->assertIsArray($x)` before indexing into it — do NOT reach for an inline `/** @var */` docblock.
- **[convention]** A release bumps THREE things in lockstep: `src/Version.php` `VERSION` const, a matching `## [x.y.z]` heading in `CHANGELOG.md` (VersionTest asserts the heading exists for the current VERSION), and an annotated git tag `vX.Y.Z` pushed after merge. `composer.json` has NO `version` field — git tags are what drive `detain/phlix-shared:^x.y` resolution via the consuming repo's composer VCS repo, so tagging is required before a downstream `composer update` can pick the change up.
- **[gotcha]** Local `master` in this workspace can silently diverge from `origin/master` (seen: 24 phantom local commits ahead AND 24 behind, plus a stale stash). Before starting work, `git fetch origin` and check `git merge-base --is-ancestor HEAD origin/master`. To start a clean branch off the real tip: `git stash -u`, `git checkout -B <branch> origin/master`, `git stash pop`. To resync local master after a merge when `git pull` says "Not possible to fast-forward", use `git reset --hard origin/master`.
- **[pattern]** phlix-shared has no `.git/hooks/pre-commit` (caliber hook absent) and gitignores `.caliber/`, `coverage.xml`, `coverage-report/` — commit directly in the live repo with explicit per-file `git add`; no throwaway clone needed.
- **[pattern]** Local verification gates before pushing phlix-shared: `./vendor/bin/phpunit` (or `composer test`), `composer stan` (PHPStan L9), `composer cs` (PHPCS PSR-12), and for releases `./vendor/bin/psalm` (errorLevel 1) + `composer validate --strict`. Schema JSON edits should also be smoke-validated with `php -r 'json_decode(file_get_contents("schemas/x.json"), ...)'`.
- **[pattern]** Schema additions here are kept additive/backward-compatible (new optional properties, widened enums) and the `### Added`/`### Changed` CHANGELOG entry documents the consuming surface (which controller/SPA reads it) — match this style.
